Learn CSS

What is CSS?

Cascading Style Sheets (CSS) is a stylesheet language that describes the presentation of various markup languages like HTML and SVG.

CSS is a rules-based language. Rules look like this:

      selector {
        property1: value1;
        property2: value2;
      }

Selectors tell the browser which nodes to apply styles to in the DOM.

Properties specify how to style the node.

Values can be strings, colors, px, rem, numbers, %, or other types depending on the property type. It's important to understand how CSS works to apply styles to the DOM.

It's not very important to memorize lots of the selectors and style properties. You'll naturally build up a knowledge of your most commonly used selectors and style properties over time with practice.

Try It!

Edit this page to include the following CSS inside the "style" block:

      body {
        margin: 0 auto;
        width: 95%;
        max-width: 440px;
        background-color: mintcream;
      }

The page should now be looking a bit more centered. We've added a centering technique using the "margin" property (more on how that works later), a "width" value of "95%" (95% of the width of "body"s parent node), a "max-width" of "440px" (440 pixels), and a "background-color" of "mintcream" (...delicious!).

Visual Heirarchy

Let's tune up the font to make the visual heirarchy more clear. On this page, the header elements "h1" and "h2" are the most important in the visual heirachy. You can increase/decrease an elements position in the visual heirarchy by modifying its size, spacing, color, background, to make the elment more or less visually interesting. "Visual heirarchy" is the design principle of arranging elements in the interface in a way that illuminates the order of importance.

Squint test: squint your eyes to illuminate the visual heirarchy of the page. Even though the content becomes a bit blurry it should remain clear what the major parts of the page are.

Let's make the "h1" font "sans-serif" instead of the default "serif".

Add this css rule to the "style" block:

      h1 {
        font-family: sans-serif;
      }

Do the "h2" elements too.

      h2 {
        font-family: sans-serif;
      }

Looks like we're repeating ourselves a bit...we can condense two rules into one by using a selector list

Replace the two rules for h1 and h2 with this single rule:

      h1, h2 {
        font-family: sans-serif;
      }

Now let's make the CSS pop out a bit:

     pre {
      padding: 16px;
      border-radius: 0.5rem;
      border: dashed 2px lightblue;
      font-family: monospace;
    }

The complete style block should look like this now:

        /* CSS goes here */
        body {
          margin: 0 auto;
          width: 95%;
          max-width: 440px;
          background-color: mintcream;
        }
        h1,
        h2 {
          font-family: sans-serif;
        }
        pre {
          padding: 16px;
          border-radius: 0.5rem;
          border: dashed 2px lightblue;
          font-family: monospace;
        }
